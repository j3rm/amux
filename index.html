amux

Run dozens of Claude Code agents unattended.
Python 3 + tmux. No build step, no external services.

Install GitHub Docs 
$ git clone https://github.com/mixpeek/amux && cd amux && ./install.sh
$ amux register api --dir ~/Dev/api --yolo && amux start api
$ amux serve  # → https://localhost:8822
  • Self-healing watchdog — a background thread snapshots every session every 60s. Context below 20% → sends /compact. Thinking-block corruption detected in output → restarts and replays the last message. Session stuck waiting for input → auto-responds. All silent, all broadcast as toast alerts.
  • Truly headless YOLO mode — --dangerously-skip-permissions still shows blocking tool-approval prompts. amux detects them by matching the "Esc to cancel" marker in terminal output (that string never appears on open-ended questions) and auto-answers, keeping sessions actually unattended.
  • Agent coordination out of the box — $AMUX_SESSION and $AMUX_URL are injected at every session's startup. A global memory file shipped with the full REST API reference lets agents discover peers, peek their output, post tasks to the board, and delegate work — without being explicitly programmed to do so.
  • Atomic task claiming — no broker — POST /api/board/:id/claim uses SQLite 3.35+ UPDATE … RETURNING as a compare-and-swap: the row only moves todo → doing if it's still unclaimed. Dozens of agents can race for the same queue with no Redis, no queue broker, no duplicated work.
  • Web dashboard + PWA — live session cards with status, token stats, and quick-actions. Peek mode with full scrollback, search, and file previews. Multi-pane workspace to watch agents side by side. Kanban board with iCal sync. Installable on iOS and Android. Tailscale gives you real TLS and phone access with zero config.
  • One Python file, zero build step — ~12k lines: a ThreadingHTTPServer with manual routing plus the entire dashboard as inline HTML/CSS/JS. Edit the file; it restarts itself via os.execv on save. Only dependencies are python3 and tmux.
