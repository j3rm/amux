Conversation Fork

Clone any session's conversation to a new branch point. Two agents explore different approaches from the same starting state.

How it works

  1. Open the session menu (⋯) and choose Clone / Fork
  2. amux starts the clone with --resume <id> --fork-session flags
  3. The fork gets a new conversation ID while sharing the prior history
  4. Fallback: if no JSONL, captures last 3000 lines of scrollback as context

When to use this

  • Two agents need to try different approaches to the same problem
  • You want to continue work on a new git branch without losing context
  • Recovering from a corrupted session by forking from a good checkpoint

Example

# Fork via API
curl -sk -X POST $AMUX_URL/api/sessions/myproject/clone \
  -H 'Content-Type: application/json' \
  -d '{"new_name":"myproject-v2"}'
